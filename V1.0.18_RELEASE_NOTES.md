## Configuration

### Basic Example

```json
{
  "_id": "c-105",
  "_parentId": "b-10",
  "_type": "component",
  "_component": "fadetext",
  "_classes": "",
  "_layout": "full",
  "title": "Scroll to Reveal",
  "displayTitle": "Scroll to Reveal",
  "body": "Your text content here. Each word will fade in as you scroll.",
  "_fadeText": {
    "_triggerPoint": 0.6,
    "_fadedColor": "#cccccc",
    "_activeColor": "#000000",
    "_transitionSpeed": "0.3s",
    "_smoothness": "ease-out",
    "_disableAnimation": false
  }
}
```

### Configuration Options

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `body` | String | Required | The text content that will fade |
| `_triggerPoint` | Number | 0.6 | Viewport position where fade begins (0-1) |
| `_fadedColor` | String | #cccccc | Color before fade-in |
| `_activeColor` | String | #000000 | Color after fade-in |
| `_transitionSpeed` | String | 0.3s | CSS transition duration |
| `_smoothness` | String | ease-out | CSS timing function |
| `_disableAnimation` | Boolean | false | Disable animation for accessibility |

## What Was Fixed in This Release

### v1.0.18 - Duplicate Body Field Fix
**Problem**: Authoring Tool showed two Body text fields  
**Cause**: properties.schema defined body field twice (inherited + explicit)  
**Solution**: Removed explicit body definition, use inherited field only  
**Result**: ‚úÖ One Body field in Authoring Tool

### v1.0.17 - Duplicate Text Display Fix
**Problem**: Body text appeared twice in compiled courses  
**Cause**: Template used `{{> component this}}` partial + custom body rendering  
**Solution**: Removed component partial, manual header rendering only  
**Result**: ‚úÖ Body text appears once

### v1.0.16 - JavaScript Runtime Fix
**Problem**: TypeError: this.checkIfResetOnRevisit is not a function  
**Cause**: Method from Adapt v4 doesn't exist in Adapt v5  
**Solution**: Removed deprecated method call from preRender()  
**Result**: ‚úÖ No JavaScript errors

### v1.0.15 - LESS Compilation Fix
**Problem**: NameError: .respond-to-small-device is undefined  
**Cause**: Adapt Framework mixins not available in all versions  
**Solution**: Replaced with standard CSS @media queries  
**Result**: ‚úÖ Course compiles successfully
